Timeline

A project for Stanford University CS225A: Experimental Robotics

  • April 2015

    The Start of a Journey

    Team Kuka Pong is formed and begins planning and building a ping pong playing robot.

  • May 14, 2015

    Milestone One: Basic Functionality

    After initial testing and development, we will have a tracking system for the ping pong ball, results from our controller, and the KUKA should be able to intersect the trajectory.

    Goals

    • Track ball position using Omnitrak vision system outputting position and velocity

    • Filter and smooth the position data

    • Predict ball trajectory

    • Move end effector with paddle to intercept the ball (orientation of paddle will not be accounted for yet)

  • May 29, 2015

    Milestone Two: Minimal Functioning Demo

    We will have the KUKA return the ping pong ball from any part of the court.

    Goals

    • Orient paddle for desired swing trajectory

    • Achieve desired swing velocity at impact

    • Create state machine to simplify control logic and for easier incremental development

    • Return ball to desired position

  • June 4, 2015

    Final Demonstration and Experimental Features

    We will have a working demo where the Kuka robot will be able to play ping pong with a human. Some experimental features may include shot selection, human tracking, and a victory dance

    Goals

    • Reset robot to neutral position

    • Successful volley (multiple returns)

  • Final

    Completed

Preliminary Controller

The controller for Milestone 1 will be a task space controller with a position gain to act on the error between current position and the desired intercept point. These desired coordinated will be updated continuously as the ball’s predicted trajectory is updated. A required velocity will be calculated based on the predicted time of intercept between the paddle and the ball, which will be monitored by a damper to keep the velocity from exceeding hardware limits.


Controller Tests in Simulation

Milestone 1

  • Move the kuka to a default position where the base and end effector are perpendicular and facing away from the ping pong table

  • Manually input coordinates of hypothetical ball positions and have the end effector reach the desired position without running into singularities or unstable motions

  • Input an XY trajectory and move the end effector to reach the trajectory within a certain time

  • Add a Z component to the trajectory affected by gravity and have the end effector reach a point on the new trajectory within a certain time

Milestone 2

  • Move kuka end effector to coordinates with a specified orientation throughout the entire path. (paddle should always be perpendicular to the table)

  • Move end effector to reach coordinates at a specified time with a goal velocity
